# Setup Guide

This document describes how to install, configure, and run the Risk-Sensitive Deep Hedging project locally.

For the project overview and feature list, see **README.md** at the repository root. For the full phased experiment-to-script map, see **Source Code/REPLICATION.md**.

All executable code lives inside the `Source Code/` folder. The commands below assume you are working from there unless noted otherwise.

---

## 1. Prerequisites

Before installing, make sure the following are available on your machine.

| Requirement | Minimum version | Notes |
|-------------|-----------------|-------|
| Operating system | Windows 10 / macOS 12 / Ubuntu 20.04 or newer | All scripts tested cross-platform |
| Python | 3.10 or newer | 3.11 recommended |
| pip | 23.0 or newer | Bundled with modern Python installs |
| Git | 2.30 or newer | For cloning the repository |
| RAM | 8 GB minimum, 16 GB recommended | Training is RAM-bound at large batch sizes |
| Disk | 5 GB free | Results, cached market data, and model checkpoints |
| GPU (optional) | CUDA 11.8 or newer with PyTorch 2.x | Strongly recommended for full training runs |
| Network | Required on first run | yfinance downloads SPY and VIX history |

The project does not use any database, frontend framework, or hosted service.

## 2. Installation Steps

### 2.1 Clone the repository

```
git clone https://github.com/chauhansiddharth1203/Risk-sensitive-deep-hedging.git
cd Risk-sensitive-deep-hedging
```

### 2.2 Enter the Source Code folder

```
cd "Source Code"
```

All `pip` and `python` commands below run from this directory.

### 2.3 Create and activate a virtual environment

**Windows (PowerShell):**

```
python -m venv venv
venv\Scripts\Activate.ps1
```

**Windows (CMD):**

```
python -m venv venv
venv\Scripts\activate.bat
```

**macOS or Linux:**

```
python3 -m venv venv
source venv/bin/activate
```

### 2.4 Install Python dependencies

```
pip install --upgrade pip
pip install -r requirements.txt
```

Do **not** use `sudo pip install`. Always install inside the virtual environment.

If you have a CUDA-capable GPU and want GPU acceleration, install the CUDA build of PyTorch instead of the default. Visit https://pytorch.org/get-started/locally/ for the install command matching your CUDA version, run it inside the same venv, then run `pip install -r requirements.txt` to add the remaining dependencies.

### 2.5 Verify the install

```
python -c "import torch; print('PyTorch', torch.__version__, 'CUDA:', torch.cuda.is_available())"
python -c "import yfinance; print('yfinance OK')"
python -c "import scipy, pandas, numpy, matplotlib; print('numerical stack OK')"
```

All four commands should print success lines.

## 3. Environment Variables Required

This project does not require any secret environment variables. The market data source (yfinance) does not need an API key.

The following optional environment variables can be set to adjust default behaviour. None are mandatory.

| Variable | Purpose | Default |
|----------|---------|---------|
| `PYTHONHASHSEED` | Reproducibility across runs | Set to `0` to match report figures exactly |
| `CUDA_VISIBLE_DEVICES` | Restrict which GPU is used | All GPUs visible by default |
| `MPLBACKEND` | matplotlib backend | `Agg` recommended for headless servers |
| `TQDM_DISABLE` | Disable progress bars in CI | Not set by default |

To set a variable for a single run:

**Windows (PowerShell):**

```
$env:PYTHONHASHSEED = "0"
python main_heston_benchmark.py
```

**macOS or Linux:**

```
PYTHONHASHSEED=0 python main_heston_benchmark.py
```

No `.env` file is used. No secrets, tokens, or credentials are committed to the repository.

## 4. Database Setup Instructions

This project does **not** use a database. All data is either:

* Generated by simulation (Heston and Bates paths, multi-asset processes)
* Downloaded on demand from Yahoo Finance via `yfinance` (SPY daily closes, VIX history)
* Cached locally under `Source Code/data/` after first download

No database server, connection string, schema, or migration is required.

If you delete the `Source Code/data/` folder, the next run will re-download the required market history and rebuild the local cache automatically.

## 5. Commands to Run the Project

All commands below run from inside the `Source Code/` folder with the virtual environment activated.

### 5.1 Headline simulated result (about 1 minute on CPU)

```
python main_heston_benchmark.py
```

Output: `results/heston_benchmark.png`. Stock+VS deep hedger should exceed the analytical Heston delta-vega ceiling.

### 5.2 Three central thesis claims (about 20 to 40 minutes on CPU)

```
python main_heston_benchmark.py
python main_path_dependent.py
python main_misspecification.py
python main_robust_training.py
```

These four scripts reproduce the evidence for Claims (i), (ii), and (iii) summarised in Section 13 of the report.

### 5.3 Real market capstone (regime-gated architecture)

```
python main_vix_gated_multiseed.py --thr-init 3.0 --freeze-gate
```

This reproduces the final positive result of the project: the regime-gated VIX architecture that closes the 2020 COVID crisis gap while preserving the 2024 calm-market win, across all five random seeds.

### 5.4 Minimum replication of all three claims (about 2 to 4 hours on CPU)

```
python main_heston_benchmark.py
python main_misspecification.py
python main_robust_training.py
python main_backtest_training.py --epochs 200 --lam 0.5
python main_vix_futures_v2.py
python main_vix_gated_multiseed.py --thr-init 3.0 --freeze-gate --epochs 200
```

### 5.5 Full report regeneration

The complete phased run sequence (Phases A, B, C, D and the optional post-report extensions) is documented in `REPLICATION.md` sections 3 through 6.

To reproduce every figure used in the report:

```
python make_report_plots.py
python make_threshold_sweep_plot.py
```

Consolidated figures are written to `results/` and `results/report/`. Do not rename image files because the LaTeX source references them by exact filename.

### 5.6 Pre-built report PDF

The final report PDF is included at the repository root as `DDP Final Report CH21B103.pdf` (56 pages). The LaTeX source used to build it is preserved in the author's working directory and is not bundled with this submission package.

## 6. Common Issues and Fixes

| Issue | Fix |
|-------|-----|
| `yfinance` download fails | Retry; check network; delete stale cache in `data/` |
| CUDA out of memory | Pass `--device cpu` where supported, or reduce `--N` |
| VIX training oscillates | Use `main_vix_futures_v2.py` (tighter action bounds, lower learning rate) |
| Different CVaR decimals | Expected across PyTorch versions; check ranking, not exact match |
| Missing `results/` images | Run the corresponding `main_*.py` first |
| `pip install` fails on Windows for `torch` | Use the CUDA-specific install command from pytorch.org instead of `requirements.txt` line |

## 7. Security and Submission Notes

This repository follows the department submission checklist.

* No passwords, API keys, tokens, certificates, or other secrets are committed
* All dependencies are pinned to specific versions in `Source Code/requirements.txt`
* `requirements.txt` versions can be regenerated with `pip list --format=freeze > requirements.txt` after a clean install
* All git operations should use a Personal Access Token (PAT) or SSH key, not embedded credentials in remote URLs
* No `sudo` is required in any installation or run command
* No services are exposed by default; the codebase runs entirely as a local Python process
